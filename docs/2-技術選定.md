# 技術選定

[ Frontend (Web) ]
↓ HTTPS
[ Backend API (sunabar) ]
↓
[ 銀行提供API ]

## 🖥 フロントエンド（Web）

### 技術

- **Next.js**
- **TypeScript**
- **Tailwind CSS**
- **fetch or TanStack Query（余裕があれば）**

### 理由

- 4人開発で**型があるのは必須**
- 画面：
    - 入金総額表示
    - 貯金／消費／投資の割合スライダー
    - シミュレーション結果表示
- Next.jsなら
    - 画面遷移が楽
    - API呼び出しが整理しやすい

👉 **フロントは「表示と操作だけ」**

👉 銀行APIは**絶対に直接呼ばない（重要）**

---

## 🧠 バックエンド（sunabar）

### 技術

- **Node.js + Express**
- **TypeScript**
- **axios（銀行API呼び出し用）**
- **dotenv（環境変数）**

### 理由

- sunabarと相性が良い
- Expressは最短でAPIを切れる
- 銀行APIキーを**サーバー側に隠せる**

---

## 🔐 セキュリティ設計

### 役割分担を明確に

| 層 | 役割 |
| --- | --- |
| フロント | UI・入力・結果表示 |
| バック | 認証・銀行API呼び出し・集計 |
| 銀行API | 残高・入金履歴・口座振り分け |

---

## 🏦 銀行APIの使い方

### 使用API

- 入出金明細照会 API(GET)
- 残高照会 API(GET)
- つかいわけ口座振り分け API(POST)
- 口座一覧照会API(GET)

### バックエンド処理例

1. 入金履歴取得
2. 割合（貯金・お小遣い・投資）を適用
3. 結果をフロントへ返却
